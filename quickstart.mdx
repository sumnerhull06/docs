---
title: "Get Started with the RepairDesk Authorize.Net Bridge"
sidebarTitle: "Quickstart"
description: "Clone the bridge, set your RepairDesk and Authorize.Net credentials, and process your first hosted checkout payment in under 10 minutes."
---

This guide walks you through cloning the bridge, wiring up your credentials, and processing a real payment session from start to finish. By the end you will have a running bridge, a completed Authorize.Net checkout, and a payment recorded in RepairDesk — all in under ten minutes.

<Steps>
  <Step title="Clone and install">
    Clone the repository and install its Node.js dependencies.

    ```bash
    git clone https://github.com/sumnerhull06/Repairdesk-api.git
    cd Repairdesk-api
    npm install
    ```
  </Step>

  <Step title="Create your environment file">
    Copy the example file and open it in your editor.

    ```bash
    cp .env.example .env
    ```

    Fill in at minimum these seven variables:

    ```ini title=".env"
    # Public URL where this bridge is reachable (used in redirect URLs)
    APP_BASE_URL=https://bridge.yourshop.com

    # RepairDesk Public API v1 base URL
    REPAIRDESK_BASE_URL=https://api.repairdesk.co/api/web/v1

    # RepairDesk API key (omit if using OAuth instead)
    REPAIRDESK_API_KEY=your-repairdesk-api-key

    # The RepairDesk payment method ID that represents Authorize.Net
    REPAIRDESK_PAYMENT_METHOD_ID=your-payment-method-id

    # Use 'sandbox' while testing; switch to 'production' when you go live
    AUTHORIZE_NET_ENV=sandbox

    # Authorize.Net API login ID
    AUTHORIZE_NET_API_LOGIN_ID=your-api-login-id

    # Authorize.Net transaction key
    AUTHORIZE_NET_TRANSACTION_KEY=your-transaction-key
    ```
  </Step>

  <Step title="Start the bridge">
    For local development, run:

    ```bash
    npm run dev
    ```

    For a production deployment, build first and then start:

    ```bash
    npm run build && npm start
    ```

    When the server is ready you will see:

    ```
    RepairDesk bridge listening on port 3000
    ```

    You can confirm the bridge is healthy at any time:

    ```bash
    curl http://localhost:3000/health
    # {"ok":true}
    ```
  </Step>

  <Step title="Create your first payment session">
    Send a `POST` request to `/api/payments/sessions` with the invoice details.

    ```bash
    curl -X POST http://localhost:3000/api/payments/sessions \
      -H "Content-Type: application/json" \
      -d '{
        "invoiceId": "12345",
        "amount": 99.99,
        "customerEmail": "customer@example.com",
        "customerFirstName": "Alex",
        "customerLastName": "Smith"
      }'
    ```

    The bridge creates a local session, requests a hosted payment token from Authorize.Net, and responds with HTTP 201:

    ```json
    {
      "sessionId": "sess_a1b2c3d4",
      "invoiceId": "12345",
      "amount": 99.99,
      "authorizeNetToken": "<hosted-payment-token>",
      "checkoutUrl": "http://localhost:3000/checkout/sess_a1b2c3d4",
      "statusUrl": "http://localhost:3000/api/payments/sessions/sess_a1b2c3d4"
    }
    ```
  </Step>

  <Step title="Open the checkout">
    Open the `checkoutUrl` in a browser. The bridge renders a launcher page that immediately posts the hosted payment token to Authorize.Net, and Authorize.Net's own checkout page appears. The customer enters their card details directly on Authorize.Net's servers — card data never passes through the bridge.

    <Tip>
      During development you can also use the shortcut launch URL to skip the API call entirely:
      `GET /launch?invoiceId=12345`
      The bridge creates the session server-side and redirects straight into the hosted checkout.
    </Tip>
  </Step>

  <Step title="Verify the payment">
    After the customer submits payment, Authorize.Net fires a webhook to your bridge, which records the payment in RepairDesk and marks the session as `recorded`. Poll the `statusUrl` to confirm:

    ```bash
    curl http://localhost:3000/api/payments/sessions/sess_a1b2c3d4
    ```

    A successfully recorded session looks like this:

    ```json
    {
      "sessionId": "sess_a1b2c3d4",
      "invoiceId": "12345",
      "amount": 99.99,
      "status": "recorded",
      "transactionId": "auth-net-txn-id",
      "recordedAt": "2024-01-15T14:32:00.000Z"
    }
    ```
  </Step>
</Steps>

<Note>
  The steps above use the sandbox environment and do not require a live webhook. For production, you must configure Authorize.Net to POST webhook events to `POST /webhooks/authorize-net` on your publicly reachable bridge URL. See the **Webhook Setup** guide for full instructions.
</Note>
