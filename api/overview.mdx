The RepairDesk Authorize.Net Bridge exposes a small REST API for creating payment sessions, checking session status, and receiving Authorize.Net webhook events. All endpoints accept and return JSON unless otherwise noted — the `/launch` endpoint and a handful of browser-facing routes return HTML pages instead.

## Base URL

```
https://your-bridge.example.com
```

Set `APP_BASE_URL` in your `.env` file to the public URL where your bridge is reachable. The bridge embeds this value into `checkoutUrl` and `statusUrl` fields in API responses, and into the return and cancel URLs it registers with Authorize.Net when creating a hosted payment page.

## Authentication

The bridge API does not require authentication of its own — it is designed as a server-to-server or staff-side service. Protect the bridge behind a firewall or reverse proxy if you do not want it publicly accessible.

The one exception is the `/webhooks/authorize-net` endpoint: when `AUTHORIZE_NET_SIGNATURE_KEY` is configured, the bridge validates the Authorize.Net `x-anet-signature` HMAC-SHA512 header on every incoming request. See the [webhook reference](/api/webhooks) for details.

<Note>
  Set `AUTHORIZE_NET_SIGNATURE_KEY` in your `.env` file to enable webhook signature verification. When the key is set, requests with a missing or invalid signature receive a `401 Unauthorized` response. If the key is not set, the bridge accepts all incoming webhook requests without verification.
</Note>

## Content type

All `POST` requests must include the `Content-Type: application/json` header. The webhook endpoint is an exception — Authorize.Net sends `application/json` automatically, and the bridge reads the raw body for signature verification before parsing.

## Additional endpoints

The following endpoints are part of the bridge but do not have dedicated reference pages. They are browser-facing and operate on HTML responses.

| Method | Path | Description |
|---|---|---|
| `GET` | `/checkout/:sessionId` | Renders the Authorize.Net hosted checkout launcher page for an existing session. Used by `checkoutUrl` links returned from `POST /api/payments/sessions`. |
| `GET` | `/payments/return` | Return URL that Authorize.Net redirects the customer to after a completed or attempted payment. Accepts a `sessionId` query parameter and renders a polling status page. |
| `GET` | `/payments/cancel` | Cancel URL that Authorize.Net redirects the customer to when they click the cancel button. Marks the session as `canceled` and renders a confirmation page. |
| `GET` | `/oauth/repairdesk/callback` | OAuth callback handler for RepairDesk authorization. Displays the returned authorization code so it can be exchanged for API tokens during initial setup. |

## Health check

```
GET /health
```

Returns `{"ok": true}` when the bridge is running. No authentication is required. Use this endpoint to verify your deployment is reachable before creating payment sessions.

```json
{ "ok": true }
```

## Error responses

All API errors return a JSON object with a single `error` string field describing what went wrong. Browser-facing endpoints (`/launch`, `/checkout/:sessionId`, `/payments/return`, `/payments/cancel`) return an HTML error page instead of JSON.

```json
{ "error": "invoiceId is required" }
```

| Status code | Meaning |
|---|---|
| `400 Bad Request` | Missing or invalid request parameters (e.g. `invoiceId` not provided, amount exceeds invoice balance) |
| `401 Unauthorized` | Webhook signature verification failed |
| `404 Not Found` | The requested resource does not exist (e.g. unknown `sessionId`) |
| `500 Internal Server Error` | An unexpected error occurred during webhook processing |
| `502 Bad Gateway` | An upstream API call to RepairDesk or Authorize.Net failed |
